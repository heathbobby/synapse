# Cursor Agent Bootstrap (Project)

This file is generated by the orchestration framework bootstrap to configure agents working in this project.

## Runtime locations (do not commit)

- `.orchestration/runtime/agent-sync/`
- `.orchestration/runtime/iterations/`

## Configuration (commit-able)

- `.orchestration/config/`
  - `framework.yaml` (project orchestration settings)
  - `workflows/` (workflow YAML configs)

## Command shorthand

Read and follow:

- `.orchestration/runtime/agent-sync/COMMAND_SHORTHAND.md`
- Fallback if runtime bootstrap files have not been generated yet:
  `orchestration-framework/templates/COMMAND_SHORTHAND.md`

## Synapse artifact factory

- Workflow: `.orchestration/config/workflows/synapse-artifact-factory.yaml`
- Personas: `.orchestration/config/agent-personas.yaml`
- Runbook: `docs/orchestration/artifact-factory-runbook.md`
- Artifact schema: `docs/artifacts/README.md`
- Default source brief: `work_items/synapse-product-brief.md`

Start from phase 0 unless a task card explicitly targets a later phase:

```text
/orchestrator::start_workflow(synapse-artifact-factory, phase-0, source-brief-normalization)
```

Keep durable strategy artifacts under `docs/artifacts/`. Keep generated task
cards, memos, iteration contexts, and agent logs under `.orchestration/runtime/`.

## Default workflow loop

1. Orchestrator generates tasks:
   - `/orchestrator::start_workflow(<workflow>, <phase>, <iteration>)`
2. Agents execute tasks (manual or via Cursor CLI):
   - `agent -p ...` (see `https://cursor.com/docs/cli/overview`)
3. Agents post `ready-to-consume` memos.
4. Integrator converges:
   - `/integrator::apply_ready(...)`
